Windows 里 viper 取 GOPATH 环境变量失败的解决办法

Windows 里 viper 取 GOPATH 环境变量失败的解决办法

问题

  使用 viper 取配置信息时，取 $GOPATH/src/dbconfig 目录下的配置文件，由于 Window 和 Linux 系统下的通配符不一样，用 runtime.GOOS 判断是 Window 或者 Linux ，然后使用分支分别填入配置信息的路径。代码如下：

        Os := runtime.GOOS
    // 获取系统类型，win 系统用 %，linux 用 $
    if Os == "windows" {
        //viper.AddConfigPath("d:/golang/src/dbconfig")
        viper.AddConfigPath("/%GOPATH%/src/dbconfig")
        fmt.Println("windows")

    } else {
        viper.AddConfigPath("$GOPATH/src/config")
        fmt.Println("Linux")
    }

 以上代码会出现异常，异常如下：

windows
read DB error: %v Config File "db" Not Found in "[d:\\%GOROOT%\\src\\dbconfig]"
db type is :%s , db string is : %s  host= user= dbname= port= sslmode= password=
`` is not officially supported, running under compatibility mode.
panic: sql: unknown driver "" (forgotten import?)

goroutine 1 [running]:

 网上搜索 win7、win8、win10 %GOPATH 等关键词会出来一堆的 gopath 配置环境的文章。但是解决不了问题。使用 go env 查了环境数据，显示是正确的。

set GOHOSTOS=windows
set GOOS=windows
set GOPATH=D:\golang
set GOPROXY=
set GORACE=

  突然灵光一闪，既然 go env 是正常的，通过 os 包取 gopath 不也一样么？马上动手试试。果然，问题解决了！

Os := runtime.GOOS
    // 获取系统类型，win 系统用 %，linux 用 $
    if Os == "windows" {
        viper.AddConfigPath(os.Getenv("GOPATH") + "/src/dbconfig")
        fmt.Println("windows")

    } else {
        viper.AddConfigPath("$GOPATH/src/config")
        fmt.Println("Linux")
    }

总结

  windows 里 go 的环境变量可以使用 os.Getenv() 取出来。如果有其他方法，欢迎留言给我，谢谢。
